# Contributing to GitMaster Dashboard

First off, thank you for considering contributing to GitMaster! It's people like you that make GitMaster such a great tool.

## How Can I Contribute?

### Reporting Bugs
* Check if the bug has already been reported in the Issues section.
* If not, create a new issue with a clear title and description.
* Include steps to reproduce the bug and your operating system.

### Suggesting Enhancements
* Open an issue with the tag `enhancement`.
* Explain why this feature would be useful to most users.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the app runs locally without errors.
4. Issue that pull request!

## Style Guide
* Follow PEP 8 for Python code.
* Use Tabler UI components for any new frontend features.
* Keep the "Portable" nature of the app in mind (everything should work from a single folder).
